# text_processing_repository
Графематический и морфологический анализ русского текста
Описание проекта

Данный проект представляет собой поэтапный пайплайн лингвистической обработки русскоязычного текста на материале дневниковых записей.
Цель проекта — продемонстрировать основные этапы предварительной обработки текста и морфологического анализа с использованием популярных Python-библиотек для компьютерной лингвистики и NLP.

Проект выполнен в учебно-исследовательском формате и охватывает сразу несколько подходов к анализу текста: от очистки и удаления стоп-слов до лемматизации, стемминга и присвоения грамматических тегов.

Основной функционал

Проект реализует следующий конвейер обработки текста:

Очистка текста (text cleaning)

Удаление стоп-слов

Лемматизация

PyMorphy3

spaCy

Стемминг (SnowballStemmer)

Морфологическая разметка

Stanza

PyMorphy3

Каждый этап сохраняет результат в отдельный файл, что упрощает анализ и отладку.

Используемые библиотеки

clean-text — нормализация и очистка текста

nltk — токенизация, стоп-слова, стемминг

pymorphy3 — морфологический анализ и лемматизация

spaCy — лемматизация с использованием готовой модели

razdel — токенизация русскоязычного текста

stanza — POS-теггинг и морфологическая разметка

Структура обработки
1. Очистка текста

На первом этапе выполняется нормализация текста:

исправление Unicode

перевод в нижний регистр

удаление переносов строк

удаление URL, email, чисел, пунктуации и эмодзи

Результат:
Дневник_экспедиции_без_эмоджи.txt

2. Удаление стоп-слов

Используются русские стоп-слова из nltk

Токенизация реализована через регулярные выражения

Удаляются все служебные слова, не несущие смысловой нагрузки

Результат:
Дневник_экспедиции_без_стоп-слов.txt

3. Лемматизация
PyMorphy3

Морфологический анализ каждого токена

Получение нормальной формы слова

Результат сохраняется в виде списка лемм

Файл:
Дневник_экспедиции_lemmas_pymorphy3.txt

spaCy

Используется модель ru_core_news_sm

Формат вывода: токен → лемма

Файл:
Дневник_экспедиции_lemmas_spacy.txt

4. Стемминг

Реализован с помощью SnowballStemmer

Токенизация — библиотека razdel

Демонстрируется отличие стемминга от лемматизации

Файл:
Дневник_экспедиции_стемминг.txt

5. Морфологическая разметка
Stanza

Для каждого токена сохраняются:

исходная форма

лемма

часть речи (UPOS)

грамматические признаки (FEATS)

Файл:
Дневник_экспедиции_tags_stanza.txt

PyMorphy3

Подробный морфологический анализ:

начальная форма

часть речи

граммемы (в кириллическом представлении)

Файл:
Дневник_экспедиции_pymorphy_анализ.txt

Результаты каждого шага сохраняются в файл → удобно для анализа и отчётности
